# Screenshot Specifications

Complete device screenshot requirements for App Store Connect.

## iPhone Screenshots

### iPhone 6.7" Display (Required)
**Devices:** iPhone 16 Pro Max, 15 Pro Max, 14 Pro Max
- Portrait: 1290 × 2796 px
- Landscape: 2796 × 1290 px

### iPhone 6.5" Display (Alternative)
**Devices:** iPhone 14 Plus, 13 Pro Max, 12 Pro Max, 11 Pro Max, XS Max
- Portrait: 1242 × 2688 px OR 1284 × 2778 px
- Landscape: 2688 × 1242 px OR 2778 × 1284 px

### iPhone 5.5" Display (Optional, for older device support)
**Devices:** iPhone 8 Plus, 7 Plus, 6s Plus
- Portrait: 1242 × 2208 px
- Landscape: 2208 × 1242 px

## iPad Screenshots

### iPad Pro 12.9" Display (Required if supporting iPad)
**Devices:** iPad Pro 12.9" (all generations)
- Portrait: 2048 × 2732 px
- Landscape: 2732 × 2048 px

### iPad Pro 11" Display (Alternative)
**Devices:** iPad Pro 11", iPad Air (4th gen+)
- Portrait: 1668 × 2388 px
- Landscape: 2388 × 1668 px

## Apple Watch Screenshots (if applicable)

## App Preview Videos

### Specifications
- Length: 15-30 seconds
- Format: H.264, M4V, MP4, or MOV
- Frame rate: 30 fps
- Audio: Optional, AAC stereo

### iPhone Video Dimensions
- 6.7": 1290 × 2796 or 886 × 1920
- 6.5": 1242 × 2688 or 886 × 1920

### iPad Video Dimensions
- 12.9": 2048 × 2732 or 1200 × 1600

## Screenshot Count

- **Minimum:** 1 per device class
- **Maximum:** 10 per device class
- **Recommended:** 5-7 (sweet spot for telling app story)
- **App Previews:** Maximum 3 per device class

## Content Strategy

### Screenshot Order (by importance)
1. **Hero shot** - Most compelling feature or main value proposition
2. **Core feature 1** - Primary use case demonstration
3. **Core feature 2** - Secondary major feature
4. **Unique differentiator** - What sets app apart
5. **Additional features** - Supporting capabilities

### Do's
- Show real app UI in use (with realistic data)
- Use text overlays to explain benefits
- Maintain consistent visual style and branding
- Localize text for each language
- Show diverse content where applicable
- Feature dark mode if supported

### Don'ts
- Show empty states or onboarding screens
- Include status bar with personal info
- Use actual user data
- Mix orientations inconsistently
- Use low-resolution images
- Show error states

### Caption Best Practices
- Position: Top or bottom, consistent across set
- Length: 3-6 words maximum
- Focus: Benefits over features
- Font: Readable at thumbnail size (minimum 60pt recommended)
- Contrast: Ensure readability against background

## iMessage App Screenshots

Same dimensions as iPhone, required if app has iMessage extension:
- 6.7": 1290 × 2796 px
- 6.5": 1242 × 2688 px

Show the iMessage app in context of a conversation.

## App Clip Screenshots

### Card Image
- 1920 × 640 px
- Shows in App Clip card when invoked
- Should represent the specific clip experience

## File Format Requirements

- **Format:** PNG or JPEG
- **Color space:** sRGB or P3
- **Bit depth:** 8-bit
- **No alpha/transparency:** Must have solid background
- **No rounded corners:** Apple adds these automatically

# Pre-Submission Checklist

Validate all requirements before submitting to App Store Review.

## Metadata Checklist

### Required Fields
- [ ] App Name (≤30 characters)
- [ ] Subtitle (≤30 characters)
- [ ] Description (≤4000 characters)
- [ ] Keywords (≤100 characters, comma-separated)
- [ ] Support URL (valid, publicly accessible)
- [ ] Copyright (format: © YEAR Company Name)
- [ ] Primary category selected
- [ ] Content rating completed

### Optional but Recommended
- [ ] Promotional Text (≤170 characters)
- [ ] Marketing URL
- [ ] Secondary category (if applicable)

### Version-Specific
- [ ] What's New text for this version
- [ ] Version number incremented correctly
- [ ] Build number unique

## Screenshot Checklist

### iPhone (Required)
- [ ] 6.7" display screenshots uploaded (1290 × 2796)
- [ ] Minimum 1, recommended 5-7 screenshots
- [ ] First 3 screenshots tell compelling story
- [ ] No placeholder or test content visible

### iPad (if Universal app)
- [ ] 12.9" display screenshots uploaded (2048 × 2732)
- [ ] Consistent style with iPhone screenshots

### Apple Watch (if WatchOS support)
- [ ] Appropriate watch size screenshots

### Quality Checks
- [ ] No status bars with personal info
- [ ] Text readable at thumbnail size
- [ ] Consistent visual style across all
- [ ] Localized for each supported language

## App Review Information

### Sign-In Apps
- [ ] Demo account username provided
- [ ] Demo account password provided
- [ ] Account is active and accessible
- [ ] Account has data to demonstrate features

### Review Notes
- [ ] Special instructions documented
- [ ] Hardware requirements explained (if any)
- [ ] Location-based features explained
- [ ] In-app purchase test guidance (if applicable)

### Attachments
- [ ] Demo video uploaded (for complex flows)
- [ ] Additional documentation (if needed)

## Content Compliance

### App Store Guidelines Quick Check
- [ ] No placeholder content
- [ ] No broken links or features
- [ ] All in-app purchases functional
- [ ] Privacy policy URL valid (if collecting data)
- [ ] Age rating accurate
- [ ] No hidden features
- [ ] Login works for reviewers
- [ ] Push notification permission has clear purpose

### Common Rejection Reasons to Avoid
- [ ] App is complete and functional
- [ ] Description matches actual features
- [ ] No crash on launch
- [ ] All claimed features accessible
- [ ] Payment flows work correctly
- [ ] No references to other platforms (Android)
- [ ] No beta/test language in visible UI

## Build Checklist

- [ ] Built with latest stable Xcode
- [ ] Tested on physical devices
- [ ] Tested on oldest supported iOS version
- [ ] No compiler warnings (recommended)
- [ ] Symbols included for crash reporting
- [ ] Bitcode included (if required)
- [ ] App thinning variants validated

## Privacy & Legal

- [ ] Privacy policy URL valid
- [ ] App Privacy details completed in App Store Connect
- [ ] Data collection accurately described
- [ ] Third-party SDK disclosures included
- [ ] IDFA usage declared correctly (ATT if tracking)
- [ ] Terms of service linked (if applicable)

## Localization Checklist

For each supported language:
- [ ] App Name localized
- [ ] Subtitle localized
- [ ] Description localized
- [ ] Keywords localized
- [ ] What's New localized
- [ ] Screenshots localized
- [ ] Promotional Text localized (if used)

## Final Submission Steps

1. [ ] All metadata saved in App Store Connect
2. [ ] Build uploaded via Xcode or Transporter
3. [ ] Build selected for this version
4. [ ] Export compliance answered
5. [ ] Content rights confirmed
6. [ ] Advertising ID usage declared
7. [ ] Submit for Review clicked
8. [ ] Release method selected (manual/automatic)

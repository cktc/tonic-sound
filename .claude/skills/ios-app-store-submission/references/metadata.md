# Metadata Reference

Complete reference for all App Store Connect text fields with examples and templates.

## Character Limits Summary

| Field | Limit | Notes |
|-------|-------|-------|
| App Name | 30 chars | Cannot change frequently |
| Subtitle | 30 chars | Shows below name in store |
| Promotional Text | 170 chars | Updatable without new version |
| Description | 4000 chars | Requires new version to update |
| Keywords | 100 chars | Comma-separated, no spaces |
| What's New | 4000 chars | Per-version release notes |
| Copyright | 256 chars | Format: "Â© 2024 Company Name" |
| Support URL | Required | Must be valid, working link |
| Marketing URL | Optional | Landing page or website |

## Keywords Strategy

### High-Value Keyword Patterns
- Action verbs: track, manage, edit, create, organize
- Problem terms: fix, solve, automate, simplify
- Audience terms: professional, personal, family, team
- Category alternatives: if photo app, try: pictures, images, pics

### Keywords to Avoid
- Words in your app name (already indexed)
- Apple trademarked terms
- Competitor names (rejection risk)
- Category name (assigned separately)
- Generic terms: app, free, best

### Example Keyword Sets

**Fitness App:**

## App Clip Metadata

### Subtitle (max 56 chars)
Should describe what the App Clip does in context of invocation.

Examples:
- "Order ahead and skip the line"
- "Rent a bike in seconds"
- "Check in without the app"

### Action Button Text
Choose from: Open, View, Play, Use, Order, Buy, Rent, Reserve, Book

### Header Image Requirements
- Size: 1920 Ã— 640 pixels
- Format: PNG or JPEG
- Content: Show app UI or branded imagery
- Text: Minimal, readable at small sizes

## URL Requirements

### Support URL
- Must be publicly accessible
- Should have clear contact method
- Can be: support page, FAQ, contact form, email link

### Marketing URL (optional)
- Product landing page
- Company website
- App-specific microsite
